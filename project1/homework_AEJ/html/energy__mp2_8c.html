Detailed Description

MP2 (Møller–Plesset second-order) correlation energy calculation.

This program reads molecular orbital information and two-electron integrals from a TREXIO file and computes the MP2 correlation energy for a closed-shell system (restricted reference).

The code:

  • Opens a TREXIO file (h2o.h5) in read-only mode.
  • Reads the number of spin-up electrons to determine the number of occupied MOs.
  • Reads the total number of molecular orbitals and determines the number of virtual orbitals.
  • Reads MO energies.
  • Reads two-electron integrals (ERI) stored in sparse format from TREXIO.
  • Filters and stores only the required subset of integrals of type (occ,occ|virt,virt).
  • Computes the MP2 correlation energy using the canonical MP2 expression.

int main ( )

Main function to compute the MP2 correlation energy.

Workflow: 1) Open the TREXIO file. 2) Read electron and orbital information:

  • number of spin-up electrons (n_up) -> occupied orbitals for closed-shell
  • number of molecular orbitals (mo_num)
  • MO energies 3) Read the sparse list of two-electron integrals (ERI). 4) Store only the (occ,occ|virt,virt) block into a dense tensor G(i,j,a,b). 5) Compute the MP2 correlation energy using the canonical MP2 formula. 6) Print the result, free memory, and exit.
Returns
int Returns 0 upon successful execution.

Number of spin-up electrons.

Initialized to a physically impossible value to make it obvious if reading from the file fails or does not overwrite the variable.

Number of occupied orbitals for a closed-shell reference.

For restricted closed-shell systems, n_occ equals n_up.

Number of virtual orbitals.

Virtual orbitals are those above the occupied space: n_virt = mo_num - n_occ.

Array of molecular orbital energies.

The array is allocated with length mo_num and filled from TREXIO.

Number of non-zero ERIs stored in the TREXIO file (sparse format).

Allocate memory for sparse ERI storage.

The integrals are stored as:

  • index: integer quadruples (i,j,k,l), stored consecutively
  • value: corresponding floating-point integral values

Read ERIs from TREXIO in a single batch.

offset_file selects the starting point in the file (here 0), and buffer_size indicates how many integrals will be read.

Allocate the dense MP2 tensor G(i,j,a,b).

Only the occupied-occupied / virtual-virtual block is stored: i,j run over occupied orbitals, and a,b over virtual orbitals.

Storage is in a 1D array of length n_occ*n_occ*n_virt*n_virt.

Populate G tensor from the sparse ERI list.

Each integral is read as (i,j,k,l)=value. The indices are canonicalized to enforce permutational symmetry, and only integrals of the form: (occ,occ|virt,virt) are stored.

Virtual indices are shifted to start at 0: a = k - n_occ, b = l - n_occ.

Close the TREXIO file after reading all required data.

Macro for convenient access to G(i,j,a,b).

a and b are virtual indices in the range 0..n_virt-1.

Compute the MP2 correlation energy.

Loops:

  • i,j over occupied orbitals
  • a,b over virtual orbitals

Denominator: eps_i + eps_j - eps_a - eps_b

Numerator: (ij|ab) [ 2(ij|ab) - (ij|ba) ]

Print the final MP2 correlation energy.

Free allocated memory to prevent leaks.
